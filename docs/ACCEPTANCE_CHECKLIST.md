# Story Forge 开发前验收清单

更新时间：2026-07-04

每次新 Codex 窗口继续开发前，先按此清单确认当前基础功能没有破坏。

## 基础运行

- [ ] 项目可以启动。
- [ ] 首页可以打开。
- [ ] `npm run build` 可以通过。
- [ ] 页面没有明显中文乱码。
- [ ] 页面没有明显横向溢出或模块重叠。

## 推荐主题

- [ ] 推荐主题显示 5 条。
- [ ] 后台候选主题总数为 30 条。
- [ ] 点击“刷新主题”后仍显示 5 条推荐主题。
- [ ] 点击“用此主题生成”可以创建任务。

## 当前任务

- [ ] 当前任务显示最新 5 条。
- [ ] 最新任务在最上方。
- [ ] 当前任务可以清空。
- [ ] 单个任务可以删除/取消。
- [ ] 删除或清空后的任务刷新页面不会重新出现。
- [ ] Writer / QA / Rewrite 轮次显示正确。

## 作品库

- [ ] 作品库从本地 `stories/` 文件夹读取。
- [ ] 作品库分页正常。
- [ ] 作品库按 `created_at` 倒序。
- [ ] 没有 `story.meta.json` 的文件夹不会展示。
- [ ] 没有 `final.md` 的文件夹不会展示。
- [ ] `story.meta.json.type !== "story"` 的文件夹不会展示。
- [ ] `story.meta.json.status !== "done"` 的文件夹不会展示。
- [ ] `story.meta.json` 可读写。
- [ ] `read_count` 可保存。
- [ ] `drop_off_users` 可保存。
- [ ] 保存后刷新页面仍显示最新指标。
- [ ] 可以打开整个 `stories/` 文件夹。
- [ ] 可以打开单篇作品文件夹。

## 页面结构

- [ ] 页面不展示“今日创作策略”模块。
- [ ] 页面右上角不展示旧版“刷新”按钮。
- [ ] 作品库标题显示为“作品库”。
- [ ] 右侧栏不展示大量 missing 文件状态。
- [ ] 左侧导航文案符合中文用户习惯。
- [ ] 左侧导航点击后选中底色变化。
- [ ] 左侧导航点击后页面滚动到对应模块。
- [ ] 页面下滑时左侧导航仍可见。

## 架构约束

- [ ] 没有重构 Story Manager。
- [ ] 没有新增 Agent。
- [ ] 没有修改核心 Workflow。
- [ ] 没有新增 TopicPlannerAgent、DiversityAgent 或 RankingAgent。
- [ ] 没有把 UI 改成纯 mock。
- [ ] 没有把作品库主数据源改成数据库。
- [ ] 没有接番茄后台。
- [ ] 没有自动发布逻辑。
- [ ] 没有把真实 API key 写入仓库。

## 文档同步

- [ ] `docs/HANDOFF.md` 与当前项目状态一致。
- [ ] `docs/CURRENT_SPEC.md` 与当前产品规格一致。
- [ ] `docs/TEST_PLAN.md` 覆盖基础功能。
- [ ] 如发现实现与文档不一致，已在交接文档中标注。
