# 格式规则

## 章节标题

- 使用 Markdown 二级标题。
- 格式为 `## 第 N 章 标题`。
- 章节标题应简短、有场景或冲突感。

## 正文段落

- 每段建议 1 到 4 句。
- 移动端阅读优先，避免单段超过 120 个汉字。
- 动作、心理、对话可以分段，保持节奏清晰。
- 不使用过多空行制造篇幅。

## 对话格式

- 中文对话使用中文引号。
- 每个说话人换段。
- 对话前后配合动作或神态，但避免每句都写“他说”“她说”。
- 对话必须推进信息、关系或冲突。

## 标点与排版

- 使用中文标点。
- 数字可按语境使用阿拉伯数字或中文数字，但同一类表达保持一致。
- 不在正文中插入作者说明、评分、prompt 或 QA 注释。
- 不使用 Markdown 表格写正文。

## 文件约定

- 初稿固定为 `draft_v1.md`。
- 改写稿按返工轮次为 `draft_v2.md`、`draft_v3.md`。
- QA 结果按评估轮次为 `qa_v1.json`、`qa_v2.json`、`qa_v3.json`。
- 最终稿固定为 `final.md`。
- 结构化产物使用 JSON 文件，正文产物使用 Markdown 文件。
